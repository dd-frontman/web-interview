# Когда в проекте действительно нужен глобальный store, а когда можно обойтись без него?

## Ответ

Глобальный store имеет смысл использовать, когда данные должны быть доступны из разных частей приложения и синхронизированы между несколькими компонентами.

Типичные случаи:

- данные текущего пользователя;
- настройки приложения;
- глобальное состояние авторизации;
- кэш данных, используемых на нескольких страницах;
- состояние сложных UI-сценариев, например wizard или multi-step процессов.

Но если состояние используется только внутри одного компонента или небольшой группы компонентов, глобальный store не нужен. В таких случаях лучше использовать:

- локальное состояние компонента;
- composables;
- props / emits.

Частая проблема — когда store начинают использовать как универсальное хранилище для любых данных. Это увеличивает связность системы и усложняет поддержку.

Поэтому обычно применяют правило: если состояние не используется за пределами одной feature-области — оно не должно быть глобальным.
