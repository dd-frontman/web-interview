# В каких случаях вы используете Composition API и какие преимущества он даёт?

## Ответ

Composition API особенно полезен в нескольких ситуациях.

Во-первых, когда компонент содержит сложную бизнес-логику. Если компонент делает больше, чем просто отображение данных, то Options API начинает раздуваться. Composition API позволяет вынести эту логику в composables.

Во-вторых, когда логика переиспользуется между несколькими компонентами. Например:

- работа с формами;
- загрузка данных;
- pagination;
- работа с websocket;
- управление состоянием UI.

В таких случаях composable превращается в отдельный модуль логики.

В-третьих, когда нужно декомпозировать большой компонент. Даже если логика не переиспользуется, её можно разделить на composables, чтобы улучшить читаемость.

Основные преимущества:

- явные зависимости;
- лучшая переиспользуемость;
- лучшее взаимодействие с TypeScript;
- проще тестировать;
- проще читать код.

Composition API не обязательно использовать везде, но он сильно помогает, когда логика становится сложной.
