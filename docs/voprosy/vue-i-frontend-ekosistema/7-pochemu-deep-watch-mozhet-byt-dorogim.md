# Почему deep watch может быть дорогим?

## Ответ

`deep watch` отслеживает изменения не только на верхнем уровне объекта, но и внутри всей структуры данных.

Если объект большой или часто изменяется, Vue вынужден:

- обходить вложенные структуры;
- отслеживать большое количество зависимостей;
- вызывать watcher чаще, чем нужно.

Это может приводить к:

- лишним пересчётам;
- падению производительности;
- сложностям с отладкой.

Поэтому `deep watch` стоит использовать только тогда, когда действительно необходимо реагировать на изменения внутри всей структуры.

Во многих случаях лучше:

- следить за конкретным полем;
- использовать `computed`;
- делать иммутабельные обновления.
