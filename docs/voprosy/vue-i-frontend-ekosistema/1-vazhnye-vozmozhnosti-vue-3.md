# Какие возможности Vue 3 вы считаете наиболее важными в реальной разработке?

## Ответ

В Vue 3 наиболее важные изменения связаны не столько с новыми API, сколько с улучшением архитектурной модели разработки.

Первое ключевое изменение — Composition API. Он позволяет организовывать код по логике, а не по lifecycle-хукам. В Options API логика одного сценария часто была разбросана между `data`, `methods`, `computed`, `watch`, что усложняло поддержку. Composition API позволяет объединять связанную функциональность в один модуль.

Второе важное изменение — новая система реактивности. Vue 3 использует Proxy-based реактивность, что позволяет:

- корректно отслеживать изменения объектов и массивов;
- убрать ограничения Vue 2 (`Vue.set`, проблемы с index в массивах);
- упростить работу с реактивными структурами.

Третье — лучшее взаимодействие с TypeScript. Vue 3 проектировался с учётом типизации, поэтому:

- props проще типизируются;
- composables легко описывать через generics;
- store и API-контракты становятся безопаснее.

Также важны:

- Teleport для модалок и порталов;
- Suspense для async-компонентов;
- улучшения производительности;
- более современный tooling, например Vite.

В реальной разработке больше всего ценится именно структурирование логики через composables и улучшенная реактивность.
