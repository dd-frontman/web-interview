# Какие ошибки чаще всего допускают разработчики при работе с реактивностью во Vue?

## Ответ

Есть несколько типичных ошибок.

Первая — мутация `props`. Props должны рассматриваться как read-only данные.

Вторая — потеря реактивности при деструктуризации. Если деструктурировать `reactive`-объект напрямую, реактивность может потеряться.

Третья — избыточные watchers. Иногда `watch` используют там, где лучше подходит `computed`.

Четвёртая — `deep watch` без необходимости.

Пятая — сложная логика внутри шаблонов. Когда вычисления происходят прямо в template, это ухудшает читаемость и может влиять на производительность.

Лучшей практикой считается:

- держать вычисления в `computed`;
- использовать composables для сложной логики;
- минимизировать side-effects внутри компонентов.
