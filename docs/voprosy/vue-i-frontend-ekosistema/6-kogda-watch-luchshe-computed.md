# В каких случаях watch лучше computed, а в каких нет?

## Ответ

`computed` используется для производных данных.

Он подходит, когда значение можно получить чистой функцией от других реактивных значений.

Например:

- фильтрация списка;
- вычисление статуса;
- подготовка данных для UI.

Главное преимущество `computed` — кеширование. Он пересчитывается только тогда, когда изменяются зависимости.

`watch` используется для побочных эффектов.

Например:

- выполнение API-запроса;
- запись в `localStorage`;
- логирование;
- запуск анимации;
- синхронизация с внешними системами.

Если логика изменяет внешнее состояние или выполняет действия — это обычно `watch`.

Если логика просто вычисляет новое значение — это `computed`.
