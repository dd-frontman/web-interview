# Как вы проверяете, соответствует ли код архитектуре проекта?

## Ответ

Обычно для этого я проверяю несколько вещей.

Во-первых, структуру файлов:

- правильно ли размещён компонент;
- соответствует ли модуль своей предметной области.

Во-вторых, границы слоёв.

Например:

- UI не должен напрямую работать с API;
- бизнес-логика не должна находиться в шаблоне;
- store не должен содержать случайные side-effects.

В-третьих, использование общих решений.

Если в проекте уже есть:

- composables;
- сервисы;
- утилиты;

лучше использовать их, чем писать новый код с нуля.

Это помогает сохранять единообразие проекта.
