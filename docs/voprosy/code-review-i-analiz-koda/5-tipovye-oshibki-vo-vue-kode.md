# Какие типовые ошибки во Vue-коде вы чаще всего замечаете на code review?

## Ответ

Есть несколько типичных проблем.

## Слишком большие компоненты

Компонент начинает содержать:

- UI;
- бизнес-логику;
- API-запросы;
- управление состоянием.

В таких случаях код сложно читать и тестировать.

## Сложная логика в template

Например:

- длинные выражения;
- сложные условия;
- вычисления прямо в шаблоне.

Лучше переносить такие вещи в `computed`.

## Избыточные watchers

Иногда разработчики используют `watch` там, где лучше подходит `computed`.

Это делает код менее предсказуемым.

## Потеря реактивности

Частая ошибка — деструктуризация `reactive`-объектов без `toRefs` или `storeToRefs`.

Это приводит к тому, что данные перестают обновляться.

## Использование index как key

Это может приводить к проблемам при обновлении списков.

Лучше использовать стабильные уникальные идентификаторы.
