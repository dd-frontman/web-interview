# Как вы проводите code review?

## Ответ

Code review для меня — это не просто проверка синтаксиса, а механизм контроля качества системы.

Обычно я смотрю на изменения в несколько этапов.

Сначала проверяю контекст задачи:

- соответствует ли решение требованиям;
- не добавляет ли оно лишнюю сложность;
- решает ли проблему полностью.

Затем анализирую архитектурную часть:

- соответствует ли код архитектуре проекта;
- не нарушает ли границы модулей;
- не появляется ли лишняя связность между компонентами.

После этого смотрю на реализацию:

- читаемость кода;
- правильность реактивности;
- обработку ошибок;
- типизацию;
- потенциальные проблемы с производительностью.

И в конце проверяю поддерживаемость:

- понятны ли имена переменных;
- не дублируется ли логика;
- можно ли упростить решение.

Главная цель ревью — не просто найти ошибки, а сделать код более понятным и устойчивым к изменениям.
