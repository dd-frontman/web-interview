# Как вы формулируете замечания на code review?

## Ответ

Я стараюсь формулировать замечания конструктивно.

Обычно я:

- объясняю проблему;
- предлагаю возможное решение;
- объясняю, почему это важно.

Например, вместо "это неправильно" лучше написать:

> в этом месте может возникнуть проблема с реактивностью, потому что объект деструктурируется напрямую. Возможно, стоит использовать `toRefs`, чтобы сохранить реактивность.

Такой подход помогает не только исправить код, но и сделать ревью полезным для всей команды.
