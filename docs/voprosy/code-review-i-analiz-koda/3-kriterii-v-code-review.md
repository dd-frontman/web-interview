# Какие критерии для вас самые важные в code review?

## Ответ

Обычно я оцениваю код по нескольким критериям.

## 1. Корректность

Код должен решать задачу без ошибок и учитывать пограничные случаи.

## 2. Архитектурная согласованность

Код должен соответствовать принятым в проекте паттернам.

Если каждый разработчик пишет по-своему, проект быстро становится трудно поддерживать.

## 3. Читаемость

Код должен быть понятен другому разработчику.

Если решение требует длительного анализа, это уже сигнал, что его можно упростить.

## 4. Поддерживаемость

Важно понимать, насколько легко будет изменять этот код в будущем.

Например:

- можно ли добавить новую функциональность;
- не придётся ли переписывать всё заново.

## 5. Производительность

Не всегда критично, но стоит обращать внимание на:

- лишние перерендеры;
- ненужные вычисления;
- тяжёлые операции в циклах.
