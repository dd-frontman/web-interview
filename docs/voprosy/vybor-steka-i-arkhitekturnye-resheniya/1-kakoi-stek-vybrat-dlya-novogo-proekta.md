# Какой стек вы бы выбрали для нового frontend-проекта и почему?

## Ответ

Выбор стека всегда зависит от контекста проекта, но если говорить о типичном современном веб-приложении, я бы предложил следующий базовый стек.

В качестве основного фреймворка — Vue 3, потому что он даёт хороший баланс между простотой, производительностью и возможностями архитектурной организации приложения.

Для языка — TypeScript, поскольку типизация значительно снижает количество ошибок и делает код безопаснее при рефакторинге. Особенно это важно в проектах с несколькими разработчиками и сложной бизнес-логикой.

Для сборки — Vite, потому что он значительно ускоряет разработку за счёт быстрых dev-серверов и оптимизированной сборки.

Базовый набор обычно включает:

- Vue 3;
- TypeScript;
- Vite;
- Vue Router;
- Pinia для state management.

Дальше добавляется инфраструктура качества:

- ESLint;
- Prettier;
- Husky + lint-staged;
- unit-тесты для бизнес-логики;
- e2e-тесты для критических пользовательских сценариев.

Также важно сразу продумать:

- структуру проекта;
- API-слой;
- обработку ошибок;
- мониторинг.

Потому что архитектура проекта влияет на поддерживаемость намного сильнее, чем конкретные библиотеки.
