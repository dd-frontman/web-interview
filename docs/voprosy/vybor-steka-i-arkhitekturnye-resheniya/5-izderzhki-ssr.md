# Какие дополнительные издержки появляются при использовании SSR?

## Ответ

SSR увеличивает сложность системы.

Во-первых, появляется серверный runtime, который должен выполнять рендеринг приложения.

Во-вторых, усложняется деплой:

- приложение становится гибридным;
- требуется сервер для рендеринга;
- появляется больше точек отказа.

В-третьих, возникают сложности с:

- кешированием;
- hydration;
- отладкой ошибок;
- обработкой асинхронных данных.

Также увеличиваются инфраструктурные расходы, потому что сервер должен обрабатывать больше вычислений.

Поэтому SSR имеет смысл использовать только тогда, когда преимущества действительно важны для продукта.
