# Когда TypeScript стоит подключать с самого начала, а когда допустим поэтапный переход?

## Ответ

Если проект начинается с нуля, лучше подключать TypeScript сразу.

Это даёт несколько преимуществ:

- строгие контракты между модулями;
- безопасный рефакторинг;
- лучшая поддержка IDE;
- меньше runtime-ошибок.

Но если речь идёт о существующем проекте, иногда проще вводить типизацию постепенно.

Например:

- сначала типизировать API-контракты;
- затем store;
- затем ключевые модели данных;
- потом постепенно покрывать остальной код.

Главное — чтобы `any` не превращался в постоянное решение. Типизация должна постепенно усиливаться по мере развития проекта.
