# В каких случаях для проекта достаточно CSR, а когда стоит рассматривать SSR?

## Ответ

CSR (client-side rendering) подходит для большинства внутренних систем и сложных веб-приложений.

Например:

- админ-панели;
- CRM;
- внутренние сервисы;
- dashboards;
- системы управления.

В таких приложениях:

- пользователи уже авторизованы;
- SEO не требуется;
- важнее скорость разработки и простота инфраструктуры.

SSR (server-side rendering) имеет смысл рассматривать, когда:

- важна индексация страниц поисковыми системами;
- важен быстрый первый рендер страницы;
- приложение ориентировано на широкую аудиторию.

Типичные случаи:

- маркетинговые сайты;
- интернет-магазины;
- контентные платформы;
- публичные сервисы.

Но SSR усложняет инфраструктуру, поэтому его стоит использовать только тогда, когда есть реальная бизнес-необходимость.
