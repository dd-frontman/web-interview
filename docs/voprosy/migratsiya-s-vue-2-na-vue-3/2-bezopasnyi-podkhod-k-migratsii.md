# Какой подход к миграции Vue 2 -> Vue 3 вы считаете наиболее безопасным и практичным?

## Ответ

Для живого продового проекта самым практичным я считаю поэтапный, инкрементальный подход.

Обычно стратегия выглядит так.

Сначала проводится аудит:

- какие зависимости используются;
- что несовместимо с Vue 3;
- какие части проекта наиболее рискованные;
- где больше всего legacy-паттернов.

После этого выбирается промежуточный этап, если он нужен. Например, переход через Vue 2.7 позволяет начать использовать Composition API ещё до полного переезда на Vue 3.

Дальше миграция идёт не "всё сразу", а по частям:

- сначала инфраструктурные вещи;
- затем общие компоненты;
- затем feature-модули;
- затем очистка техдолга и унификация паттернов.

Я бы избегал big bang-переезда, если только это не небольшой проект или если бизнес не готов выделить отдельное окно под полную остановку изменений.

Безопасный подход — это когда:

- есть приоритизация по риску;
- есть стратегия регрессионной проверки;
- есть временные компромиссы, но они контролируемы;
- миграция встроена в delivery, а не существует отдельно от продукта.

То есть хороший подход к миграции — это инженерный rollout, а не переписывание "с нуля ради красоты".
