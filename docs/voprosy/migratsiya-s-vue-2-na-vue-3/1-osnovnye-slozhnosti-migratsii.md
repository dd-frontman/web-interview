# Какие основные сложности возникают при миграции с Vue 2 на Vue 3?

## Ответ

Основные сложности при миграции с Vue 2 на Vue 3 обычно связаны не столько с синтаксисом, сколько с накопленным техническим долгом и несовместимостью старых паттернов с новой моделью разработки.

Во-первых, почти всегда всплывают проблемы совместимости:

- сторонние библиотеки могут не поддерживать Vue 3;
- старые плагины, фильтры или локальные хаки перестают работать;
- часть внутренних компонентов может опираться на устаревшие API.

Во-вторых, меняется сама модель организации кода. Если проект долго жил на Options API, mixins и большом количестве неявной общей логики, то переход на Composition API требует не просто переписать код, а переосмыслить его структуру.

В-третьих, часто возникает проблема с реактивностью и побочными эффектами:

- старые watchers могут вести себя не так, как ожидалось;
- deep watch начинает сильнее бросаться в глаза как источник лишней реактивной нагрузки;
- часть логики, завязанной на `this`, требует переосмысления.

Ещё один большой пласт — инфраструктурный:

- нужно пересобрать state management;
- пересмотреть типизацию;
- перепроверить подход к переиспользуемой логике;
- убедиться, что сборка, тесты и деплой не ломаются.

Главная сложность в такой миграции — не "обновить версию фреймворка", а сделать это без потери управляемости проекта и без остановки delivery.
