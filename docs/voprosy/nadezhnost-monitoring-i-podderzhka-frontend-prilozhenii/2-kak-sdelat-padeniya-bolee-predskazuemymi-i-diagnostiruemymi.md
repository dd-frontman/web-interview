# Как сделать так, чтобы падения приложения в проде были более предсказуемыми и диагностируемыми?

## Ответ

Главное правило — ошибки должны фиксироваться автоматически.

Для этого используется система error tracking, которая собирает:

- текст ошибки;
- stack trace;
- страницу, на которой произошла ошибка;
- браузер и версию устройства;
- действия пользователя перед ошибкой.

Также важно добавлять контекст:

- идентификатор пользователя, если это допустимо;
- текущий маршрут;
- состояние приложения;
- номер версии приложения.

Когда ошибка происходит, разработчик получает достаточно информации, чтобы понять:

- где именно произошла ошибка;
- при каких условиях она воспроизводится;
- какие пользователи затронуты.

Без такого механизма диагностика production-проблем становится очень сложной.
