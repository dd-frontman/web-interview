**Баблинг (event bubbling)** — это механизм распространения событий в браузере, при котором **событие, произошедшее на вложенном элементе DOM, автоматически «всплывает» вверх по дереву DOM**, последовательно вызывая обработчики на его родителях до самого корня (`document` → `window`). Это часть модели событий браузера.

---

## 🧠 Как это работает

Когда пользователь выполняет действие (например, клик), браузер генерирует событие на самом **внутреннем (целевом) элементе**, где произошёл клик, а затем **передаёт его вверх** через всех родителей, пока не доберётся до верхнего уровня:

`целевой элемент → parent → … → document → window`

Каждый из этих узлов может иметь навешанный обработчик этого события, и все они сработают при всплытии.

### Пример

HTML:

```html
<div id="parent">
    <button id="child">Click me</button>
</div>
```

JavaScript:

```js
document.getElementById("parent").addEventListener("click", () => console.log("parent clicked"));
document.getElementById("child").addEventListener("click", () => console.log("child clicked"));
```

Результат при клике на `<button>`:

```nginx
child clicked
parent clicked
```

Событие сначала обрабатывается на кнопке, затем «всплывает» к родителю.

## 📌 Почему это происходит

DOM события по умолчанию **всплывают** (bubbles) — в отличие от другой фазы, _capturing_, где события идут сверху вниз (от `document` к целевому элементу).

Формально есть **три фазы жизненного цикла события**:

1. **Capturing (перехват)** — сверху вниз
2. **Target (цель)** — на самом целевом элементе
3. **Bubbling (всплытие)** — снизу вверх  
   И по умолчанию обработчики работают именно в фазе всплытия.

---

## 🧠 Как это влияет на обработчики

**Плюсы:**  
✔ Можно назначить один обработчик на родителя вместо множества обработчиков на дочерних элементах — это называется **делегирование событий**.

✔ Упрощает работу с динамически добавляемыми элементами (новые потомки автоматически «попадают» под обработчик родителя).

**Минусы:**  
❌ Не всегда хочется, чтобы событие обрабатывалось на всех уровнях — тогда могут возникать лишние вызовы.

## 🛠 Как остановить баблинг

JavaScript позволяет **остановить дальнейшее всплытие** с помощью:

`event.stopPropagation();`

Это предотвращает передачу события выше по DOM-дереву.

## 🧩 stopPropagation, preventDefault и stopImmediatePropagation

Эти методы часто путают, потому что все они вызываются на объекте события, но решают разные задачи.

- `event.stopPropagation()` - "не сообщай родителям"
- `event.preventDefault()` - "не выполняй стандартное действие"
- `event.stopImmediatePropagation()` - "останови вообще всё дальше на этом событии"

### `event.stopPropagation()`

Останавливает дальнейшее распространение события по DOM.

Что это значит на практике:

- обработчик на текущем элементе уже выполняется;
- дальше событие не пойдёт к родителям;
- стандартное действие браузера при этом не отменяется.

```html
<div id="parent">
    <button id="child">Open</button>
</div>
```

```js
parent.addEventListener("click", () => console.log("parent"));
child.addEventListener("click", (event) => {
    event.stopPropagation();
    console.log("child");
});
```

При клике по кнопке в консоли будет только:

```text
child
```

### `event.preventDefault()`

Отменяет стандартное поведение браузера для этого события.

Примеры стандартного поведения:

- переход по ссылке;
- отправка формы;
- открытие контекстного меню;
- прокрутка страницы в некоторых сценариях.

```html
<a id="link" href="/profile">Profile</a>
```

```js
link.addEventListener("click", (event) => {
    event.preventDefault();
    console.log("go through router logic");
});
```

После этого клик по ссылке не приведёт к переходу на `/profile`, но событие всё ещё может всплыть вверх, если отдельно не вызвать `stopPropagation()`.

### `event.stopImmediatePropagation()`

Это более жёсткая версия `stopPropagation()`.

Она делает сразу две вещи:

- не даёт событию подняться выше по DOM;
- не даёт выполниться остальным обработчикам этого же события на этом же элементе.

```js
button.addEventListener("click", (event) => {
    event.stopImmediatePropagation();
    console.log("first handler");
});

button.addEventListener("click", () => {
    console.log("second handler");
});
```

Результат будет таким:

```text
first handler
```

Второй обработчик на этом же `button` уже не выполнится.

## 🧠 Как быстро запомнить разницу

- `stopPropagation()` - событие не идёт к родителям
- `preventDefault()` - браузер не делает своё стандартное действие
- `stopImmediatePropagation()` - событие не идёт дальше ни к родителям, ни к другим обработчикам на текущем элементе

## Vue: какие есть эквиваленты

Во Vue чаще всего используются такие модификаторы:

- `@click.stop` -> вызывает `event.stopPropagation()`
- `@submit.prevent` -> вызывает `event.preventDefault()`
- `@click.stop.prevent` -> комбинирует оба поведения

```html
<a href="/profile" @click.prevent="openProfile">Profile</a>
<button @click.stop="onChildClick">Click me</button>
```

Прямого шаблонного модификатора для `stopImmediatePropagation()` во Vue обычно не используют. Если это действительно нужно, такой вызов делают внутри обработчика вручную.

## 🧠 Примеры в Vue

В Vue всплытие используется внутри шаблонов так же, как и в обычном DOM. Предположим:

```html
<div @click="onParentClick">
    <button @click="onChildClick">Click me</button>
</div>
```

При клике на кнопку будут вызваны **два обработчика**:

1. `onChildClick` (на кнопке)
2. `onParentClick` (на родителе)

Это потому что событие клика **всплыло** от кнопки к родителю.

Если нужно, чтобы при клике на кнопку **не срабатывал родительский обработчик**, можно остановить всплытие:

`<button @click.stop="onChildClick">Click me</button>`

Директива `.stop` — это синтаксический сахар Vue для `event.stopPropagation()`.

---

## 🧠 Когда используют баблинг

📌 **Делегирование событий:**  
Назначают обработчик на контейнер (например `<ul>`), а не на каждый элемент списка — это экономит память и код, и позволяет обрабатывать события от динамически созданных элементов.

📌 **Глобальные слушатели:**  
Можно отлавливать события на уровне `body`/`document` и реагировать на действия внутри страницы.

---

## 📌 Итог

**Баблинг (event bubbling)** — это механизм, по которому события, сгенерированные на элементе, **«всплывают» вверх по DOM-дереву**, позволяя обработчикам на родителях реагировать на них. По умолчанию браузеры используют именно этот способ распространения событий. 🌊
