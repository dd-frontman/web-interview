# Rollup для Vue

**Rollup** — это бандлер, который особенно силён там, где нужен **чистый итоговый output** и хорошая работа с **ES-модулями**.
Во Vue его чаще выбирают не для больших SPA, а для **библиотек, UI-kit'ов и SDK**.

## Почему его ценят

- Хорошо оптимизирует библиотечные сборки.
- Даёт предсказуемый и аккуратный output.
- Сильный tree-shaking для модульного кода.
- Удобен, когда нужно собрать пакет в нескольких форматах.

## Когда особенно уместен

- Вы делаете Vue component library.
- Нужен npm-пакет, а не полноценное приложение.
- Важен маленький размер артефакта и контроль над форматом выхода.
- Нужно экспортировать ESM и другие форматы сборки.

## Что важно понимать

- Для Vue есть плагины обработки `.vue`-компонентов, но экосистема вокруг app-development уже не такая удобная, как у Vite.
- Для девелоперского workflow большого приложения Rollup обычно не первый выбор.
- Зато он отлично чувствует себя как инструмент для library mode.

## Ограничения

- Менее удобен как центральный инструмент для повседневной разработки большого Vue SPA.
- Часть задач, которые в Vite решаются "из коробки", здесь требуют более явной настройки.

## Короткий вывод

Если вы собираете **библиотеку на Vue**, Rollup часто выглядит лучше, чем Webpack.
Если вы делаете **приложение**, то обычно удобнее смотреть в сторону Vite.
