# Vite для Vue

**Vite** в экосистеме Vue обычно является **дефолтным выбором для новых проектов**.
Он даёт очень быстрый локальный старт, удобный HMR и простой конфиг.

## Почему его часто выбирают

- Очень быстрый dev-server за счёт ESM и лёгкой обработки модулей.
- Хорошо работает с Vue Single File Components через официальный плагин.
- Для production-сборки даёт tree-shaking, code splitting и нормальную работу с ассетами.
- Обычно требует меньше конфигурации, чем Webpack.

## Когда особенно уместен

- Новый Vue 3 SPA-проект.
- Админка, кабинет, внутренний продукт, маркетинговый сайт.
- Команда хочет быстро стартовать без тяжёлого build-конфига.
- Нужны Vitest, modern DX и простой plugin pipeline.

## Что важно понимать

- В dev-режиме Vite делает ставку не на классический полный bundle, а на быстрый модульный сервер.
- В production он остаётся полноценным инструментом сборки и оптимизации.
- Для большинства Vue-проектов его возможностей хватает без кастомного low-level тюнинга.

## Ограничения

- Если проект глубоко завязан на старую webpack-экосистему, миграция может быть небыстрой.
- Очень специфичные legacy-loader цепочки иногда проще поддерживать в Webpack/Rspack.

## Короткий вывод

Если вопрос звучит как **"что взять для нового Vue-проекта?"**, то обычно ответ: **Vite**.
Он даёт лучший баланс между скоростью разработки, простотой и зрелостью экосистемы.
